# POS 工作流第五轮联调测试

## 测试目的
验证 POS 工作流全链路自动化流程：Issue → 分支 → 提交 → PR → 审核 → 合并，全程无人工干预。

## 测试步骤
1. 创建 Issue（标题注明轮次）
2. 从 main 创建 feat 分支
3. 提交测试文件到分支
4. 创建 PR（关联 Issue）
5. ArkClaw 审核（必须先读 PR 实际内容）
6. 审核通过后合并 PR + 删除分支

## 预期结果
- Forge 自动完成步骤1-4，输出 PR 链接
- ArkClaw 自动审核，给出基于实际内容的结论
- 合并后分支自动清理
